because.client package¶

Base components for implementing HTTP clients.

Each subpackage in because.interfaces, like “qt” or “python”, offers its own flavor of Client (which may need its own flavor of SSLConfig, returns its own flavor of Transfer, etc.) This package contains base classes that are reused across those flavors.

If you just want to use services to get data, you probably don’t need anything from this subpackage, and may find it confusing.

If you are implementing a flavor or trying to understand how existing things work, you will find this subpackage to be important.
