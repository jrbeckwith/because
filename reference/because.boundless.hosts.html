because.boundless.hosts module¶

Definitions for publicly accessible hosts running BCS services.

This module contains only a dictionary called HOSTS. Each key is a short alias for a host giving its role, e.g. “dev”, “test”, or “prod”. Each value is an instance of boundless.client.service.Host, containing a base URL and any associated information necessary to use it, aside from information about services and endpoints.
