class because.client.request.Request(method, url, body=None, headers=None)[source]¶

Bases: because.base.utils.PrettyPrintable

Represent parameters for making an HTTP request.

You may be saying to yourself: “why yet another class Request?!?” Well – this internal representation is used to abstract over representations like QNetworkRequest, so that because can work about the same in multiple environments without all the response processing code having to be duplicated with minor variations in it.

Excludes some important operational parameters, like SSL configuration and timeout duration, that do not really specify the request payload. This also does not represent a request as it is being performed. That’s a Transfer.
