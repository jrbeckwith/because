class because.interfaces.qt.transfer.Transfer(request, ssl_config=None, log=None, _send=None)[source]¶

Bases: because.transfer.Transfer

Hold per-request state for one ongoing HTTP request cycle.

This is also used as a device for bundling up multiple descriptions of a transfer’s state to be handled by one callback, instead of requiring many separate callbacks to handle many different state notifications.

The method is offered and blocks to honor the base class contract.

because you are likely e.g. to make the GUI thread unresponsive.

class because.interfaces.qt.transfer.TransferSignals(parent=None)[source]¶

Bases: PyQt4.QtCore.QObject

Hold signals for an instance of interfaces.qt.Transfer.

This is the QObject handling signals for Transfer, allowing Transfer itself not to be a QObject.

Exposing these means not having to expose the QNetworkReply directly.

because.interfaces.qt.transfer.unpack_reply(reply)[source]¶

Convert a QNetworkReply to a because.response.Response.

Warning:This may destroy the state of the QNetworkReply, so don’t use the

QNetworkReply after calling this.

Warning:The returned Response may not work properly if the QNetworkReply

or objects it depends on are freed before the Response is used.

You probably want to use Transfer instead of using this. This helper function is used in the implementation of Transfer, but is exposed separately as part of the lower-level API layer.
