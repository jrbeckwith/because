Interfaces¶

Here, an “interface” is sort of a flavor or backend for using because in a particular environment.

because.interfaces.python¶

Interface using only the Python standard library.

This interface uses only the Python standard library. It is compatible with both Python 2 and Python 3. However, it uses blocking I/O.

because.interfaces.qt¶

Interface using PyQt.

This interface should work with either PyQt4 or PyQt5. It should also allow use of Qt’s signals and slots system, which also allows asynchronous operation and multithreading. However, it does not offer anything to specifically help with QGIS.

because.interfaces.qgis¶

Interface for use with QGIS Python APIs.

This is essentially the same as the qt interface, but it allows the use of QGIS facilities like the QgsNetworkAccessManager, and also provides some extras like a raster layer class for viewing XYZ basemaps based on QgsRasterLayer (based on GDAL).
